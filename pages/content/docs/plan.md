# zeroverse 実装計画

設計の全体像は [設計ドキュメント](./overview.md) を参照。本書は設計をコードに落とすための作業計画である。

## 基本方針

- **創発は連鎖であり、一段目が転ぶと全部転ぶ。** 機能単位でなく**反証可能な段階**で切る。各マイルストーンは「〜が創発するか」という検証可能な問いを持ち、答えが No のあいだは次へ進まない。
- 10000 人 × 1000 年の大規模ランは M1〜M3 が通ってから。
- ゲームとしての生命線は三点：ローカル高速シミュ（手元で 1000 年が数分）、リプレイと観戦 UI（家系図、技術伝播、板の価格チャート）、常設リーグ。

## フェーズ一覧

| フェーズ | 検証する問い | 主要成果物 |
| --- | --- | --- |
| **P0** 基盤 | （問いなし・足場） | wit/ 確定版、engine スケルトン、決定論ハーネス |
| **M1** 交易 | 自給自足より交易が有利になるか | 20 human の最小世界、生涯消費比 > 1.0 |
| **M2** 貨幣 | 特定 resource に取引が集中するか | 板（指値注文）とマッチング |
| **M3** 教育 | skill の売買が自発的に起きるか | teach/learn と if-taught-me |
| **M4** 血縁 | 血縁構造が投資行動に現れるか | 継承・conceive・sex・親密度 |
| **P5** スケール | （M1〜M3 通過後） | 大規模ラン、Shapley 採点、観戦 UI、リーグ |

## P0：基盤（実装の最初の作業）

創発の検証を始める前に、決定論とツールチェーンを固める。

### P0-1. WIT の確定

- WIT の型定義は[設計ドキュメント](./overview.md)のトピックごとの md に分散してある（骨格と分散マップは [09-wit-draft.md](./wit.md)）。各 md 末尾の「詰めるべき点」を潰して仕様を確定し、`wit/` ディレクトリに統合した確定版を起こす。
- 未定義 record を埋める：`board-quote`、`trade-info`、`teach-info`、`proposal`、`stat`、`action-kind`、`public-trade`、`give-args`。
- 型は変えてよいが「**不変の原則**」（純関数 decide / インスタンス新規化 / float 禁止 / snapshot の情報線引き / fuel→health 写像）は変えない。
- M1 では probe・conceive・teach 系は使わないが、ABI の互換性破壊を避けるため型定義だけは最初から置く。

### P0-2. Rust エンジンのスケルトン

- 単一クレート（`core`）。tick ループ、決定論的順序の `resolve()`。
- wasmtime 統合：**InstancePre + pooling allocator + fuel 計量 + NaN 正規化**。呼び出しごとに新規インスタンス化（Module 共有は可）。
- 決定論チェックリスト（[08-architecture.md](./architecture.md)）を実装で担保：
  - ABI から float 排除（qty = 1/1000 固定小数点 u64）
  - wasi を一切リンクしない
  - 乱数は human ID × tick のハッシュを snapshot で渡す
  - fuel 切れ / trap / 不正 decision は「その月は idle」に潰す
- human-id 生成は非連番（出生順を漏らさない）。

### P0-3. 決定論ハーネス（CI）

- 同一シード → バイト単位で同一の歴史、を CI で常時検証する（2 回走らせて世界状態のハッシュを比較）。
- リプレイ = シード + brain セット、のシリアライズ形式を決める。
- brain サンプル 1 種（Rust / cargo component）をビルドして engine から呼べるところまで。

**P0 の完了条件**：ダミー brain（idle のみ）で 20 human × 100 年が回り、決定論テストが CI で緑。

### 状態（2026-07: wasmtime 統合まで完了）

- `crates/wasm-host`: wasmtime（component-model）で brain component を Brain trait に載せた。呼び出しごとの新規インスタンス化・決定論 fuel 計量（fuel→health 写像が有効化）・fuel 切れ/trap の部分実行・wasi 非リンク（guest は wasm32-unknown-unknown → wasm-tools component new）。
- **scenario も wasm component 化**：`wit-scenario/scenario.wit` の init（初期条件の賦与）+ judge（クリア判定）。シナリオファイル形式の未決はこれで決着（[world.md](./world.md)）。
- guest 参照実装 `guests/`: **forager brain**（法則を知らずに生まれ、invoke の実験と invoke-result から食事を発見して生き延びる）と **scenario-m1**（全員生存 + 消費閾値でクリア判定）。`zeroverse-wasm run` で seed 42/7 はクリア、wasm 実行込みの決定論を CI で常時検証。

## M1：交易は自給自足に勝つか

最初に観測する数字は「**自給自足 human と交易 human の生涯消費の比**」。1.0 を超えるまでパラメータ（g 減衰率、T、ε、消費量）を追い込む。

### 実装項目

- world 生成：resource 5 種、組成ベクトル + 自由エネルギー g、レシピ（skill）と食事ペアの手続き的生成、resource-id / skill-id シャッフル。
- 公理の最小実装：tick（月）、寿命、能力曲線（stats）、環境循環（環境ストック、harvest のストック依存歩留まり、消費・死亡時の環境還元、環境変換と Φ 上限）、g の自然減衰、Σg_out < Σg_in の強制、**world 全体（human + 環境）での組成保存アサーション**。
- アクション：invoke（skill の発動。harvest・食事・train を包含。実験を兼ね、失敗時は材料の一部を失い理由なしの action-failed）/ give / discard / idle。skill の**獲得**（teach/learn・実験発見）は M3 なので、M1 は生得 skill のみで回す。
- 空間：総空間 S、身体 + 保有体積の占有、満杯時の保有増加制限、占有維持費 κ（health コスト）、食事・劣化の廃棄物と discard の動機付け。
- 劣化：resource 型ごとの自発変換（g 減衰の実装）、決定論的確率丸め、環境変換との適用パス統一。
- 知人：ε による偶発的出会い、introduce（triadic closure）。
- 交換：板はまだ無し。**give の相対交換のみ**（conditional-give の if-received までを M1 に入れるかは実装時に判断。入れないなら二者の give 相互発行で代替）。
- 消費と生存：health の自然減少と食事 skill による回復、fuel→health 写像（思考コスト）、初回 decide の食事ブートストラップ（baby brain 期の食事履歴を events で渡す）。
- 参照 brain 2 種：**自給自足 brain** と **交易 brain**（手書き。以後の全マイルストーンの回帰テストにもなる）。
- 計測基盤：生涯消費のロギングと集計 CLI。**生涯消費の定義は「食事など stats 獲得 invoke の Δg（入力 g − 産出 g）の生涯総和」を仮採用**（[07-scoring.md](./scoring.md)）。world 全体の組成保存・空間保存のアサーションもここに含める。

### 合格基準（反証可能）

- 交易 brain の生涯消費 ÷ 自給自足 brain の生涯消費 > 1.0 が、複数シードで安定して成立する。

### 状態（2026-07: 達成）

- `zeroverse m1` で 10/10 シード ratio 1.3〜2.5 を確認（`crates/core/src/scenarios.rs` の合格基準テストが CI で常時検証）。
- 実験系: 賦存ミスマッチ（edible ≠ 高熟練 specialty）を共通にした 4 人組（交易ペア + 同賦存の自給自足 2 人）で、群間の差を戦略のみに限定。
- 採用した仮決定: 交換は give の相互発行（conditional-give は M2 で導入）、食事ブートストラップは InvokeResult イベント（WIT への同期は wasm 統合時）、fuel→health 写像は wasm 統合時に有効化（ネイティブ参照 brain は fuel を消費しない）。

### 主要リスク

- パラメータ空間が広い。g 減衰率・invoke 歩留まり・環境変換速度・ε・消費量のスイープを自動化する（大規模ランの前哨戦でもある）。
- 環境ストックが浅すぎると 20 人でも枯渇即死し、深すぎると枯渇圧が消えて交易動機が弱まる。初期ストックと再生速度の釣り合いを最初に取る。
- 手続き的レシピ生成が「浅すぎる/深すぎる」化学を作る。resource 5 種で交易の余地（比較優位）が出る生成則を先に確認する。

## M2：貨幣は創発するか

### 実装項目

- 板：resource ペアごとの指値注文（limit-order）。記名。
- マッチング：価格優先、同価格内はシード付きシャッフル。時間優先禁止。
- 月内解決順序の固定：（M3 の teach/learn を除き）conditional-give 判定 → 板マッチング。
- snapshot に market（板の公開気配 board-quote）を追加。
- probe.trade-history（fuel 課金 pull、記名板の売買履歴）。
- 参照 brain：板を使う交易 brain。

### 合格基準

- どの resource ペアの取引よりも、**特定の一つの resource を経由する間接交換**が取引量で支配的になる world が、パラメータ調整で再現できる（＝貨幣の創発）。
- どの resource が選ばれるかが g 減衰率（貯蔵性）と整合する。

### 状態（2026-07: 達成）

- `zeroverse m2` で全シード**関与率 1000/1000 の完全収束**を確認（合格基準テストが CI で常時検証）。
- 実験系: 賦存 +2 シフト（欲望の二重の一致が構造的に存在しない）の 20 human。market brain は最初の 6 ヶ月で保有 resource の劣化を観測して媒介候補を選び、以後は公開気配の厚みへの追随（メンガーの正のフィードバック）で収束する。貨幣を指名する仕組みは world に無い。
- **予期しなかった創発**: 多くのシードで媒介に選ばれたのは **λ=0 の廃棄物**だった。腐らないが消費価値のない資産が貨幣になる——貝殻貨幣・不換紙幣と同型で、貯蔵性原理そのもの。「整合」の定式化は「選ばれた媒介の λ ≤ 最良 primary の λ」とした。
- 仮決定: 約定価格は双方の指値の**中点**（同時手番のため maker/taker が無い）、板は記名で約定相手は知人になる。conditional-give（if-received）の解決系は teach/learn と一緒に M3 で。

## M3：skill の売買は自発するか

### 実装項目

- skill：法則グラフのノードとしての skill、前提 skill 付きの実験的発見（確率的）、invoke の using-skills 判定。
- teach/learn：同月ペア成立で 1ヶ月進捗、完了まで T ヶ月（教師の熟練度と学習者の若さの関数）、両者のアクション枠消費。
- if-taught-me（standing order）：月単位アトミックな分割払い。エスクローは置かない（escrow on/off の実験パラメータ化は未決事項 5）。
- skill 漏洩の二経路：産出 resource の市場観測、教えた事実。リバースエンジニアリング（観測からの確率的逆算）。
- 月内解決順序を最終形に固定：**teach/learn 成立 → conditional-give 判定 → 板マッチング**。
- 参照 brain：教師特化 brain、徒弟 brain。

### 合格基準

- world 側に価格も契約も置かずに、skill の対価付き教育（resource 支払いを伴う teach/learn の対）が継続的に成立する。
- 秘匿（売らない・教えない）と公開（売って模倣される）の両戦略が観測される。

### 状態（2026-07: 達成）

- `zeroverse m3` で全シード確認: 徒弟 4/4 が習得、if-taught-me の月払いが約 20 回継続成立、公開教師の世界では RE（板の売りからの模倣）が発生し秘匿教師の世界では 0（合格基準テストが CI で常時検証）。
- 月内解決順序を最終形に固定: 一方向 act → teach/learn 成立 → conditional-give 判定 → 板マッチング → RE。
- 実装した機構: teach/learn の同月ペア成立（進捗 = 教師熟練% × 学習者 cognition%）、if-taught-me / if-received（2 パスで相互 OTC も成立）/ unconditional、RE（板に売りを出した resource はそれを作る skill を確率的に漏らす。primary の売り → harvest、廃棄物の売り → 食事 skill）。
- 面白い副次結果: 秘匿教師の方が月払い回数が**多い**（公開教師は RE で顧客を失う）。独占は放置しても崩れる、の数値化。
- 未実装で M3 から持ち越し: 実験による skill の確率的発見（前提 skill 付きの深い法則）。法則グラフに深さが入る「文字」実装（P5）と同時に入れるのが自然なため。

## M4：血縁は投資行動に現れるか

### 実装項目

- conceive の自動発生（相対親密度相互 50% 超・異性・fertility 窓・刷り込み判定）、出生、baby brain（0〜6歳の world 提供共通 brain）、6歳での 50/50 brain 継承（切替は観測不能）。
- sex（出生時 1/2、本人のみ可視）と出産の観測非対称（child-born は母のみ。父には 0歳知人が現れるだけ）。
- 親密度：stats 依存の月次増減 ＋ action 由来の増減、acquaintance への露出、母子の初期値。
- memory-limit の年齢関数（老化による物忘れ）、能力曲線の完成（fertility 窓）。
- 参照 brain：家族投資 brain（子に無償で teach する）、婚姻契約 brain（約束プロトコルを使う）。

### 合格基準

- 血縁度に応じた投資の傾斜（子への teach 頻度が非血縁への teach 頻度を上回る等）が、world 側に家族概念を書かずに観測される。母（血縁確実）と父（父性不確実）で投資行動に差が出るか。
- 同族内婚への閉じ込もりが skill 流入減で不利になる（外婚 = skill を買う、のトレードオフが数値に出る）。
- 婚姻契約（親密度に基づく相互の約束)が brain の戦略として維持されるか。繰り返しゲーム型で足りなければ if-intimacy（未決 #7）を入れて再実験。

### 状態（2026-07: 中核機構を達成）

- `zeroverse m4` で全シード確認: 出生 3〜6、**近親（親子・きょうだい）出生 0**（Westermarck 刷り込みが機能）、8 歳以上の子は全員が親からの無償 teach で harvest skill を獲得（血縁経由の技能伝達）、人口は有界。
- 実装した機構: pairwise 親密度（月次減衰 + give/teach/約定で増加、当事者のみ可視）、fertility の年齢窓、**conceive の自動発生**（相対親密度が相互 50% 超 + 異性 + 窓内 + 非刷り込み + 非妊娠）、妊娠・出産（health コストは母、child-born は母のみ、父には 0 歳知人が現れるだけ）、母子の高初期親密度、食事 skill の母系生得付与（#12 決定）、6 歳/18 歳での brain 切替ハーネス。
- **綺麗な創発が 2 つ**: (1) 出産後は母の親密度ポートフォリオが子に傾き、夫への相対親密度が薄まって出産間隔が自然に空く（**子への愛情が避妊になる**）。(2) M3 の教育世界でも teach の相互作用が親密度を積んで自然に出生が起きた（教師と徒弟の間に子が生まれる）。
- 持ち越し（M4 追加実験として）: 同族内婚が skill 流入減で不利になる実験、婚姻契約の維持実験（未決 #7 と表裏）、apparent-age の算出実装、母 vs 父の投資差の定量化。→ **すべて実施済み（下記）**。

### 状態（2026-07 追補: M4 派生実験を達成）

- **同族内婚 vs 族外婚**（`zeroverse m4-clans`）: 2 氏族に別々の食文化・harvest 技能を持たせ、同じ参加者・同じペア数で組み方だけを変えた。族外婚の子は harvest 技能 **1.75〜2.0 系統 / 人**、同族内婚は **1.0 系統**（全シード）。技能流入は倍増する。平均消費は 30 年・環境余裕ありの条件では同等（枯渇圧が掛かる規模で再計測が次の実験）。
- **婚姻契約の維持**（`zeroverse m4-marriage`）: 契約はエンジンに存在せず、貞節 brain は**許し付きしっぺ返し**（相手からの贈与が 6 ヶ月途絶えたら止める + 年に一度の和解の手）だけで動く。全シードで貞節ペアの出生 15〜29 vs 浮気者ペア **0**（贈与を全知人に薄く回す浮気者は誰とも相対親密度 50% を超えられない）。ペア親密度も貞節側が一貫して高い。→ **未決 #7 は導入見送りで決定**（繰り返しゲームで足りる）。
  - 実装知見: 素朴な TFT は不作の騒音で相互破綻が吸収状態になり協調が崩壊する（seed により親密度 200 → 1.2）。**年次の和解**を入れると再点火して回復する。協調の制度には「許しの周期」が要る、という創発的な教訓。
  - トークン贈与: 親密度の増分は相互作用の回数で決まり量に依らないため、軽い贈与（0.5）が合理的になる。儀礼的な贈り物の原型。
- **母 vs 父の投資差**（血縁投資台帳 `parental_investment`、`zeroverse m4` で表示）: 参照 brain は対称でも、給餌投資は母がやや優位（5 シード中 3 で母 > 父、例: seed 1 は母 375 vs 父 270）。差の源泉は**父性不確実性**: 父は「新しく現れた 0 歳の知人」を子と推測するため、ε 出会いの誤認で投資が血縁外に漏れる（台帳は実の親子だけを数える）。teach 月数はほぼ対等。
- **apparent-age を実装**: `age × (1 + β(1 − vitality))`、vitality = (health+strength)/2、β = 300‰（world パラメータ）。知人観測に実値が乗り、若さが正直なシグナルになった（human.md 更新）。snapshot の知人ビューは (id, 親密度) から **AcquaintanceView**（id・親密度・見かけ年齢・生死）に拡張。
- 次の候補: 枯渇圧下での族外婚の消費優位の検証、婚約（概念の初期 memory 接種、未決 #8）、apparent-age を使った配偶者選択 brain。

### 状態（2026-07 追補 2: sex グラデーション・見合い・三世代の再生産）

- **sex を -10〜+10 のグラデーションに変更**（[human.md](./human.md)）: 符号 1/2・二峰型（90% は |sex| 4〜10、10% が 0〜3 の曖昧域、0 = 中性）。conceive は符号が逆のペア（負側が母）。知人観測に **apparent-sex**（真値 + 観測者ペア固定ノイズ ±σ、既定 3）を追加。|apparent| が確信度になり、±2〜3 は逆符号の可能性が残る。
- **introduce を実装**: 自分の知人同士を引き合わせる（WIT には当初からあった action の engine 実装）。参照実装では親が成人した子を人脈に紹介する = **見合い**として機能し、ε 頼みでは 55 年に 1 組だった第二世代の出会いを常態化させた。
- **CourtingBrain（第二世代の求愛）**: apparent-sex の符号で相手を選び、トークン贈与の相互性でペア束縛（TFT）、数年子ができなければ見切る（誤認 = 同符号ペアの解消。真値は最後まで観測できない）。**三世代目の出生を確認**（55 年で 3 シード中 2 シード以上、良シードは 9〜10 人。受け入れテスト `third_generation_emerges`）。近親出生は 0 を維持。
- **きょうだいの共在**: 新生児は母の既存の子と出生時に知人になり共在親密度を持つ → 刷り込みが働き、見合いの導入後もきょうだい婚は忌避される（[kinship.md](./kinship.md)）。
- **研鑽（教育の上積み）**: 既習者への再教育を教師の熟練を上限に許可（[skills.md](./skills.md)）。初取得 50% で止まる第二世代は生産が薄く求愛の原資が作れなかった。
- **デバッグから出た創発の教訓が 3 つ**:
  1. **穀倉死**: λ の小さい食料を無限に採り溜めると占有維持費が食事を上回って死ぬ。備蓄には倉庫コストとの均衡点があり、参照 brain は採取スロットル（備蓄 10 で停止）を持つに至った。
  2. **親の愛が避妊になる（成人版）**: 親が成人した子に給餌を続けると、子の配偶者への相対親密度が 50% を超えられず孫ができない。**子離れ**（見かけ 18 歳以上には給餌しない）が三世代目の必要条件だった（導入前: 三世代 0〜2 / 導入後: 7〜10）。
  3. 夫婦は互いに食べられない食料を贈り合うため、**片付け（discard）は既婚者の生存条件**になる（贈答ゴミの穀倉死）。
- 未決 #6（親密度の式）・#10（刷り込みパラメータ）は据え置き。apparent-sex の σ は時代プリセット（#8）の軸に追加。

### 状態（2026-07 追補 3: 人口の持続 — 絶滅の診断と 5 つの対策）

大規模ラン（scale 10・300 年）で「世代は続くが漸減して全滅する」を診断し、構造要因を潰した。全変更後、6 シード全てで 300 年後も人口が成長を続ける（576〜1716 人）。

- **診断 1: 血縁投資が避妊だった**。conceive の相対親密度（全知人分母・相互 50%）は、出産の瞬間に母のポートフォリオを子が占有して次の妊娠を子離れ（18 年後）までブロックしていた。夫婦あたり実質 1 子 → 世代ごとに人口半減。→ **恋愛市場 share 化**: 分母を「fertility 窓内・異符号・非刷り込みの知人」に限定（閾値 40%）。
- **診断 2: 出生間隔が消えてラビットモード化**。share 化だけだと妊娠期間おきに出産し、幼子が給餌能力を超えて餓死しコホートが断絶する。→ **産後不妊**（既定 27 ヶ月。出産間隔 3 年）と**飢餓性無月経**（母 health < 40 で conceive 停止）を追加。旧方式の「ポートフォリオによる自然な間隔」を生理パラメータで置換した形。
- **診断 3: 見合いが第一世代で断絶**。introduce するのは FamilyBrain だけで、創設世代の死とともに配偶者市場が消えていた（ε は 55 年に 1 組）。→ **CourtingBrain に見合いを世代継承**。
- **診断 4: 環境が scale に連動していなかった**。人だけ n 倍すると初期から環境容量超過になり、env が半飽和点のはるか下に釘付き（採取歩留まり 7%）→ 全員栄養不良で出生が数十年停止 → 妊性世代の窓が閉じて絶滅。「大規模にすると必ず滅ぶ」の正体。→ **ExperimentSession の scale で環境（ストック・Φ・半飽和点・空間）も n 倍**し、1 人あたりの生態を scale 不変にした。
- **診断 5: 名ばかりの知人との conceive**。市場が細いと親密度 0.1 の相手でも share 1000‰ になり、「多くの子を持ち 1 人あたり養育が薄い父」（刷り込み閾値未達）と成人した娘が結ばれる近親クラスタが発生した。→ **絶対親密度の下限**（= 刷り込み閾値）を conceive に追加。未成年期に閾値を超えれば刷り込み、超えなければ減衰で下限に届かず、求愛で新たに築いた関係だけが子をなす。
- 付随する参照 brain の改良: **給餌の傾斜**（自活できない見かけ 8 歳未満に人数分をまとめ渡し。均等輪番は子 3 人超で破綻）、**求愛ターゲットの分散**（全員が知人リスト先頭を選ぶと求愛が殺到し、対称バンプで当人の分母が薄まり誰も閾値に届かない集団不妊が起きる）、FamilyBrain の食事を条件付き（health < 90）に統一。
- **実験の意味が変わった点**: 婚姻契約実験の「浮気の避妊効果」（贈与を薄めると子ができない）は share 化で消滅し、出生数は貞節/浮気でほぼ同等になった。契約の価値は**ペア親密度**（貞節 52〜184 vs 浮気 4〜24）に現れる。受け入れテストもこの規則性に更新。近親出生は「共同養育ペア = 0」を維持しつつ、非養育実父という公理上防げない希少ケース（≤1）を許容する形に精密化。

## P5：スケールと運用（M1〜M3 通過後）

- **大規模ラン**：10000 human × 1000 年のネイティブ実行。プロファイリングと並列化。
- **Shapley 採点**：ランダムな brain 組み合わせのモンテカルロ、毎ラウンド振り直し、world サイズ正規化。v(S) = Σ u(c)（凹効用）。
- **三形態デプロイ**：ネイティブ / Durable Objects（スローモー人間参加 realm）/ ブラウザ WASM（観戦・デバッグ）。
- **観戦 UI**：リプレイ（シード配布）、家系図、技術伝播、板の価格チャート。
- **常設リーグ**と brain 投稿パイプライン（LLM に brain を書かせる公式パイプライン、WIT をプロンプトとして使う）。
- **キャンペーンモード**（[world.md](./world.md)）：1 人用チュートリアル。時代・地域シナリオ + クリア条件、ビルトイン NPC brain とのブレンド、スコア = 自 brain 比率の低さ。M1〜M4 の検証シナリオ（交易・貨幣・教育・血縁）と参照 brain 群をそのまま最初のキャンペーン群・NPC に転用する。
- **追加アイデア**（[10-ideas.md](./ideas.md)）：「文字」特異ノードはこの段階以降で法則グラフに追加する。

### 状態（2026-07: ブラウザ観戦 UI の初版を達成）

- **[ビューワ](https://kuboon.github.io/zero-verse/play/)**（`pages/static/play/`、GitHub Pages 配信）: キャンペーンと brain を選択してブラウザ内で engine を実行。速度調整（1〜120 月/秒）・一時停止・1月/1年ステップ・クリア判定。human クリックで stats / 保有 / skill / 親密度 / イベントの全知インスペクタ。親密度グラフ・母子エッジ・環境ストック・板・推移チャートを可視化。
- **実装形態**: ブラウザは component model をネイティブ実行できないため、engine（`crates/web`）を wasm-bindgen で core wasm 化し、brain / scenario component は **jco transpile**（`--instantiation sync`）で core wasm + JS glue に変換して接続する（`pages/static/play/runtime.js` が wasmtime Linker 相当）。decide ごとの新規インスタンス化（テレパシー禁止）は glue 側で維持。実行は Web Worker に隔離し、応答しない brain は watchdog が worker ごと terminate する。
- **決定論**: 同一シードのブラウザ実行はネイティブ実行と **state hash が一致**する（fuel 消費が 1 health 量子未満の brain の場合。ビューワには fuel 計量が無く fuel_used = 0 のため、fuel を大きく消費する brain ではネイティブと歴史が分かれる）。公式ランはネイティブ側。
- **fuel が無いことの副作用**として forager の実験ループに無限ループを発見・修正（全組が既知になると `continue` し続ける。ネイティブでは fuel 切れが打ち切っていて、毎月 fuel 予算を全焼していた）。
- **実験再現シナリオ**（追補）: シナリオ選択に M1（交易 vs 自給自足）/ M2（貨幣の創発）/ M3（公開・秘匿教師）/ M4（家族）/ M4 派生（同族内婚・族外婚・婚姻契約）を追加。brains は zeroverse-core のネイティブ参照実装をブラウザ wasm 内で駆動し、**CLI の run_* とビルダー・月遷移・集計を共有**する（`ExperimentSession`。同一シードなら CLI と同一の state hash）。役割（交易 / 自給自足 / 教師 / 徒弟 / 貞節 / 浮気者 / 氏族 / 子）を枠線色で可視化し、📊 集計ボタンで CLI と同じサマリを任意時点で表示する。
- 持ち越し: 自作 brain（.wasm）のドラッグ&ドロップ実行（jco はブラウザ内 transpile も可能なので実現性あり）、リプレイ共有 URL（シード + 選択の埋め込み）、家系図・価格チャートビュー。
