# 血縁と婚姻

50/50 継承則（[公理 7](./axioms.md)）は一遺伝子座半数体と同型で、血縁度が厳密に出る（親子 1/2、兄弟 1/2、祖父母孫 1/4、いとこ 1/8）。Hamilton 則 rB > C が外生ルールなしで数値的に成立する。

- 血縁は **brain 同一性の唯一の（確率的）証明**である。子に教えるのは確実に自陣営投資、他人に教えるのは敵陣営強化の可能性。「家族には無償で教える」は情報の非対称性から演繹され、world 側に家族という概念を書く必要がない。
- relation-hint の廃止により、**world が血縁を証明する箇所はゼロになった**。血縁を確実に知るのは産んだ母だけ。父は conceive の時期と 0歳知人の出現タイミングの相関から確率的に推定するしかない（父性不確実性）。子は母から教わり、母を信じるしかない。母系の知識だけが確実、という非対称が投資戦略の非対称（母系相続の有利さ）として現れるはず。
- 6歳での切替は観測不能。期待血縁度が 1/2 に固定され、「外れくじの子を捨てる」戦略が原理的に実行不能。
- 両親の期待持ち分が対称に 1/2 なので共同養育が均衡になりうる。子の帰属交渉は不要になった（旧案の「婚姻を帰属交渉契約にする」は破棄。[91-rejected.md](./rejected.md) 参照）。
- 配偶者選択 = 子の半分を委ねるアルゴリズム選び。行動履歴から相手の brain の質を推定するので、協力性が配偶市場を通じて伝播する。
- 最大の抜け穴は**同族内婚**（両親同一 brain なら r = 1）。抑止は二つ：同族を確実に見つけられないこと、内婚に閉じると skill 流入が減り技術が止まること。**外婚は skill を買い、内婚は brain を守る**という戦略軸として残す。人工ペナルティは足さない。
- **marriage アクションは置かない**。conceive もアクションではない（下記）。婚姻という制度も brain が発明する。

## conceive の自動発生

conceive は**アクションではなく、親密度グラフから自動的に発生するイベント**である。

- **相対親密度**：self の全知人への親密度合計に占める、特定の相手の割合（[human.md](./human.md)）。絶対値が同じでも相対親密度は人によって異なる。
- **発生条件**：以下がすべて揃った月に conceive が発生する。
  1. 双方の相対親密度が**互いに 50% を超えている**
  2. sex の符号が逆のペア（負側が母。真値は不可視のまま、他人には apparent-sex しか見えない → [human.md](./human.md)）
  3. 双方の fertility 窓内
  4. 刷り込みペアでない（下記）
  5. 女性側が妊娠中でない
- 帰結：**求愛 = 親密度への投資**。give や teach で親密度を集中させることが配偶行動になり、「互いの親密度ポートフォリオの過半を占め合う」という定義からして排他的な関係だけが子をなす。婚姻契約の if-intimacy（[06-communication.md](./communication.md)）は、この 50% 集中を守る装置としてそのまま接続する。
- 発生しない理由は通知されない。同性か、窓外か、刷り込みか、相手の親密度が分散しているのかは推定するしかない。

### 家族愛と恋愛の区別（Westermarck 刷り込み）

相対親密度だけを条件にすると母子間で conceive が発生してしまう（新生児の親密度は母にほぼ 100% 集中する）。これを**刷り込み**で排除する。

同じ母に育てられた**きょうだいは出生時に共在する**: 新生児は母の既存の子と知人になり、共在親密度（母子初期値の半分）を持って生まれる。fertility 窓が開く前に閾値を超えるため刷り込みが働き、きょうだい婚は行動を待たずに忌避される（Westermarck の共在モデル。M4 派生で親が子を人脈に「紹介」するようになっても近親の出会いが婚姻に進まない根拠）。

- **どちらかの fertility 窓が開く前に、ペアの親密度が閾値を超えていたら「刷り込み」フラグが立つ。刷り込みペアでは conceive は永久に発生しない。**
- 母子・同居きょうだい・幼馴染が自動的に除外される。**成人後に出会った血縁は除外されない**：同族内婚の戦略軸（冒頭）は保たれ、抑止は従来どおり人工ペナルティではなく skill 流入の停滞。
- 家族愛と恋愛を 2 本の親密度チャンネルに分ける案は不採用：公理 10 の状態量が倍になり、どの action がどちらのチャンネルを動かすかという割当問題も生む。刷り込みはペアあたり 1 bit で済み、生物学的実在（Westermarck 効果）に対応する。

## 出産と観測

- **出産の観測非対称**：`child-born` は母にのみ届く。父には 0歳の知人が現れるだけで、通知はない。
- **親子の初期親密度**：母子ペアは高い初期値、それ以外（父子を含む）は 0（[human.md](./human.md) の親密度）。
- **relation-hint は廃止**。world が関係（親子・配偶者）を保証する仕組みは存在しない。spouse もシステムでは付与しない：婚姻は約束機構（[06-communication.md](./communication.md)）の上に brain が発明する。

関連イベント：`someone-died(human-id)`（[human.md](./human.md)）。

継承関連の world パラメータ（[09-wit-draft.md](./wit.md) の world-config）：`max-lifespan-months` など。

## 詰めるべき点

- [x] `relation-hint` の矛盾：**relation-hint 自体を廃止して解決**。spouse はシステム付与せず、婚姻契約は約束機構（[06-communication.md](./communication.md)）で創発させる。
- [x] conceive できる年齢範囲：fertility stat の年齢窓で表現する（[human.md](./human.md)）。窓の具体値は M4 で確定。
