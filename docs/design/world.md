# world の仕様

world = **法則 + 初期状態 + シード**。同一シードからはバイト単位で同一の歴史が再生される（[08-architecture.md](./08-architecture.md)）。world の生成もこの決定論に含まれる：生成パイプライン全体がシードからの決定的導出である。

## 生成パイプライン

world 生成はシードから次の順で導出する。各段は前段の出力だけに依存する。

1. **法則グラフの生成**
   - 組成ベクトル空間を張り、resource 種を配置する（[02-resources.md](./02-resources.md)）。
   - 各 resource に自由エネルギー g と減衰率を割り当てる。レシピは組成から手続き的に生成する。world ごとに異なる化学ができる。
   - skill ノードを法則グラフに配置する（浅い法則から深い法則へ。最深部に「文字」特異ノード → [10-ideas.md](./10-ideas.md)）。
2. **ID シャッフル**
   - resource-id と skill-id を法則グラフとの対応ごとシャッフルする。brain は試行と観測でしか法則を復元できない。
3. **world パラメータの確定**
   - Φ（自由エネルギー流入）、ε（偶発的出会い率）、g 減衰率の組、教育速度 T の係数、`act-slots-base`、`max-lifespan-months`、親密度の増減係数、apparent-age の補正係数 β など。
   - [09-wit-draft.md](./09-wit-draft.md) の world-config として brain に渡るのは brain が知ってよいもののみ。**法則グラフに関する情報は一切含まれない。**
4. **初期人口の生成**
   - 人数と年齢構成。human-id は非連番（[08-architecture.md](./08-architecture.md)）。
   - 各個体の sex（1/2）、年齢相応の stats、fertility 窓の位置。
   - 初期の知人グラフと親密度（家族単位のクラスタを作るか、全員 0 から始めるかは詰めるべき点）。
5. **初期保有の配布**
   - resource ストックと skill の配布。時代プリセット（[10-ideas.md](./10-ideas.md)）がここへの入力になる：どの深さまでの skill を誰が持って始まるか。
6. **brain の割り当て**
   - プレイヤー brain を human に割り当てる（リーグでは複数プレイヤーの brain を等分配）。
   - 初期 memory の接種：時代プリセットの「共通知識」（慣習・プロトコル）をここで入れる。

時代プリセット（狩猟 / 農耕 / 工業 / 情報）とは、このパイプラインの 3〜6 段への入力セットの名前である。

## キャンペーンモード

リーグ（Shapley 採点の対戦）とは別に、**キャンペーンモード**を置く。歴史上の特定の時代・地域を模した world に brain を送り込み、一定の条件を満たせばクリア。

- **シナリオ = 生成パイプラインへの入力一式 + クリア条件**。「地域」はゼロ次元世界ではフレーバーであり、実体は Φ・ε・法則グラフの形・初期 skill 分布・NPC 構成の組で「らしさ」を出す（例：狩猟採集の列島 = 浅い skill のみ・高 ε・低 Φ、産業革命期の都市 = 「文字」発見済み・低 ε・厚い初期板）。
- **NPC brain**：プレイヤーが送り込むのは 1〜少数の brain。残りの人口は world 側が用意する公式 NPC brain セットで埋める。NPC は時代の慣習（婚姻プロトコル等）を接種済みで、その時代の社会をだいたい再生する。
- **クリア条件**：反証可能なスカラー条件。例：
  - N 年以内に深さ d の skill を発見する（技術ツリー到達）
  - 自 brain 系譜の総消費（凹効用和）を X にする（一族繁栄）
  - 特定 resource への取引集中度を Y 以上にする（貨幣を創発させる）
  - 自 brain 系譜を Z 世代存続させる（生存）
- **判定はメタ層**：クリア判定はエンジン（観測者）が world の外側で行う。world 内のルールには一切介入しない。「human 以外の要素を world に用意しない」原則はキャンペーンでも破らない。
- **位置づけ**：リーグ参入前の brain 開発環境・チュートリアルであり、人間参加モード（スローモー realm）の入口でもある。マイルストーン M1〜M4 の検証シナリオ（交易・貨幣・教育・血縁）は、そのまま最初のキャンペーン群になる。

## 詰めるべき点

- [ ] 初期人口の血縁・知人グラフ：家族クラスタ付きで生成するか、全員孤立から始めるか。初期親密度の分布。
- [ ] 初期人口の brain：リーグでは全員プレイヤー brain だが、キャンペーンの NPC brain をどう用意するか（過去リーグの上位 brain の再利用か、公式手書きか）。
- [ ] クリア条件の記述形式：シナリオファイル（宣言的な条件記述）をリプレイ形式と揃えるか。
- [ ] キャンペーンとシード：シードが既知だと歴史を暗記した過学習 brain でクリアできてしまう。クリア判定を複数シードでの成功率にするか（[90-open-questions.md](./90-open-questions.md) #9）。
- [ ] 生成パイプライン 1 段目の具体アルゴリズム：組成ベクトルの次元数・resource 数と skill 深度の分布（[02-resources.md](./02-resources.md) の詰めるべき点と同じ問題）。
