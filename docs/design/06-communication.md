# 通信とシグナリング

- **say / message アクションは置かない**。同族認識をしたい brain は既存アクション（特定数量の贈与など）でプロトコルを自作する。
- world が無料のシグナルを提供すると詐欺師に即コピーされ無意味化する（Zahavi）。**すべての情報伝達に実コスト**（resource か時間）を持たせる。贈与数量にビットを乗せる通信は許容し、量子化の粗さ（qty の刻み）が通信帯域の価格を決める。
- プロトコルを模倣する詐欺師は、偽装利得と偽装コストが釣り合う比率で均衡する。**これは意図した仕様。**

## WIT：通信の搬送路

通信専用の型は**存在しない**のがこの設計の要点。情報は既存アクションに乗る：

- `give`（[02-resources.md](./02-resources.md)）：贈与数量にビットを乗せる。量子化の粗さ（qty の刻み 1/1000、[90-open-questions.md](./90-open-questions.md) #3）が通信帯域の価格を決める。
- `introduce` / `introduced` / `encountered`（[human.md](./human.md)）：紹介そのものがシグナル（紹介コスト = アクション枠）。
- `limit-order`（[04-market.md](./04-market.md)）：板は記名なので、注文の出し方自体が公開シグナルになる。

## 約束（コミットメント）

world は契約を執行しない。約束は次の二つの機構の組み合わせで表現する。

1. **繰り返しゲーム型**（既存機構の範囲）：conditional-give の月次連鎖。「今月◯◯があったら渡す」を毎月出し直す。徒弟制（if-taught-me）と同型。月を跨ぐ拘束力はなく、継続のインセンティブだけが担保。
2. **担保（bond）型**（検討中）：give-condition に**注文者自身の親密度を参照する条件**（if-intimacy）を足す案。「私の親密度が、あなた以外の誰かとの間で閾値 T を超えたら、あなたに X を渡す」という standing order は、**違約金付きの貞節の誓い**になる。
   - 条件が参照するのは注文者自身の状態だけ（if-taught-me と同型）なので、「自分の内側は全部見える、他人は行動の痕跡だけ」の線は破らない。発火は相手への情報開示になるが、それは担保という実コストと引き換えに自分で差し出した監視権であり、Zahavi 原則と整合する。
   - 第三者間の親密度は不可視なので、**相手を監視する契約は書けない。書けるのは自分を縛る契約だけ**。婚約とは、この自己拘束 order を互いに差し出し合う行為として定義できる。
   - world 側に契約執行を半歩入れることになるため、escrow を置かない方針（[03-skills.md](./03-skills.md)）との整合が論点（[90-open-questions.md](./90-open-questions.md) #7）。

婚姻契約とは「他の人と一定以上親密にならない」という相互の約束であり（[05-kinship.md](./05-kinship.md)）、上の機構でエンコードできるかが M4 の検証対象。プロトコルの共通知識をどう行き渡らせるかは時代プリセット（[10-ideas.md](./10-ideas.md)）が受け持つ。
