# 追加アイデア（採用方針）

いずれも採用方針だが、実装フェーズは M4 以降または未割り当て（[PLAN.md](../PLAN.md) 参照）。

## 文字

法則グラフの深部に「skill を resource に変換する」特異ノードを一つ置く。発見した文明では知識が貯蔵・取引・相続可能になり、教師ボトルネックが外れる。書物の劣化は g 減衰、印刷はさらに深部。枠組みの拡張なしで時代の相転移が出る。

## 思考コスト

brain の decide の WASM fuel 消費を、その human の health の減少に写像する。重い brain は自分の体力で遅さを払い、食事（skill）で取り戻す。限定合理性がルールでなく価格として入る。

## LLM 前提

brain の大半はプレイヤーが LLM に書かせる。自然言語の戦略書 → brain のパイプラインを公式に用意し、WIT ファイル自体をプロンプトとして機能させる。

## 時代プリセット（world 初期値）

world の初期値セットを複数用意する：**狩猟時代 / 農耕時代 / 工業時代 / 情報時代**。時代を分けるのは world のルールではなく初期条件だけ。

| 軸 | 内容 |
| --- | --- |
| 初期 skill 分布 | 法則グラフのどの深さまで発見済みか（狩猟 = 浅い skill のみ、情報 = 「文字」ノード発見済み） |
| 初期 resource | ストックの量と種類 |
| world パラメータ | Φ・ε・g 減衰率の組 |
| 初期人口 | 人数と年齢構成 |
| **初期 brain の共通知識** | プリセットの memory を接種して開始する。例：工業時代以降では human の過半数が「婚約」プロトコル（[06-communication.md](./06-communication.md) の約束機構の使い方）を知っている |

制度を world に実装するのではなく、**初期 brain の記憶に接種する**。設計原則（human 以外の要素を用意しない）とは矛盾しない：時代とは brain たちの知識状態の別名である。慣習の共通知識を初期条件にすることで、契約プロトコルのコーディネーション問題（全員が同時に同じ規約を採用しないと成立しない）を回避できる。

プリセットが world 生成のどこに入るか、およびプリセットを土台にした**キャンペーンモード**は [world.md](./world.md) を参照。
